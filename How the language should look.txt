# How Prism Should Look

Prism should be a flexible, dynamic, beginner-friendly language that feels easier than Python while still being powerful enough to grow into a serious scripting language.

The goal is not to copy Python exactly. Prism should feel like this:

- Easy to read
- Easy to write
- Dynamic by default
- Forgiving where it helps beginners
- Strict where it prevents confusing bugs
- Fast enough to become much faster than Python later
- Flexible enough to support several natural coding styles

Prism source files should use the `.pr` extension.

Example:

```pr
let name = input("What is your name? ")
output "Hello, {name}!"
```

The language should allow both simple beginner syntax and more structured advanced syntax.

---

# Core Philosophy

Prism should be designed around these ideas:

1. Simple things should be simple.
2. Beginners should not fight the syntax.
3. Dynamic typing should be the default.
4. Types should be optional, not required everywhere.
5. Common operations should have short, readable syntax.
6. The same task should usually have more than one clear way to write it.
7. Error messages should explain what happened in plain language.
8. Prism should feel friendly in scripts, REPL, games, automation, and small apps.

---

# Variables

Variables can be created with `let`.

```pr
let name = "Alex"
let age = 20
let active = true
```

Variables can also be declared without a value. The default value should be `null`.

```pr
let username
username = "Nova"
```

Constants use `const` and must always have a value.

```pr
const pi = 3.14159
```

This should raise an error:

```pr
const pi
```

Prism should also support optional type hints.

```pr
let int age = 20
let str name = "Alex"
let bool ready = true
```

But the easier dynamic style should be preferred:

```pr
let age = 20
let name = "Alex"
let ready = true
```

Possible future flexible shortcut:

```pr
name := "Alex"
age := 20
```

This could mean the same as `let name = "Alex"`.

---

# Input and Output

The core functions are:

```pr
input()
output()
```

Prism should allow normal function-call style:

```pr
output("Hello")
let name = input("Name: ")
```

Prism should also allow beginner-friendly output without parentheses:

```pr
output "Hello"
output "Name: {name}"
```

String interpolation should be easy:

```pr
let name = "Alex"
output "Hello, {name}!"
```

This should work the same as:

```pr
output(f"Hello, {name}!")
```

# Data Types

Prism should support:

- int
- float
- string
- bool
- unknown
- complex
- array
- dictionary
- set
- tuple
- null
- function
- object/class later

Prism should be dynamically typed by default.

```pr
let x = 10
x = "now I am text"
x = [1, 2, 3]
```

Optional strict mode can be added later for users who want stronger checking.

```pr
strict on
let int age = 20
age = "twenty"  # error in strict mode
```

# Numbers

Integers:

```pr
let a = 120
let b
b = 20
```

Floats:

```pr
let price = 1.20
let tax = 2.0
```

Other number literals:

```pr
let hex_value = 0xFF       # 255
let bin_value = 0b1010     # 10
let oct_value = 0o17       # 15
```

Readable numeric separators should be allowed:

```pr
let money = 1_000_000
let rate = 3.141_592
```

Complex numbers:

```pr
let a = 12j
let b = 3.5j
let c = complex(2, 5)
```

# Booleans and Unknown

Prism should support three truth states:

```pr
let yes = true
let no = false
let maybe = unknown
```

Conversions:

```pr
bool(1)        # true
bool(0)        # false
bool(-1)       # unknown
bool(true)     # true
bool(false)    # false
bool(unknown)  # unknown
```

This makes Prism useful for real-world cases where a value may be yes, no, or not known yet.

# Null

`null` means no value.

```pr
let user = null
```

A variable declared without a value should start as `null`.

```pr
let result
output(result)  # null
```

# Strings

Strings should support single, double, and triple quotes.

```pr
let a = "Hello"
let b = 'Hello'
let c = """Hello
from multiple lines"""
```

Triple quotes are for multiline strings.

Verbatim strings should use `@` and keep backslashes literally.

```pr
let path = @"C:\Users\Name\file.txt"
```

String interpolation should be first-class.

```pr
let name = "Alex"
let age = 20
output "{name} is {age} years old"
```

Strings should support:

1. Immutability
2. Indexing
3. Negative indexing
4. Slicing
5. Concatenation
6. Repetition
7. Membership checks
8. Length with `len()`

Examples:

```pr
let text = "Prism"
output text[0]       # P
output text[-1]      # m
output text[0:3]     # Pri
output "ha" * 3      # hahaha
output "is" in text  # true
output len(text)     # 5
```

String methods should include:

```pr
.upper()
.lower()
.capitalize()
.title()
.strip()
.lstrip()
.rstrip()
.find()
.index()
.replace()
.split()
.join()
.isdigit()
.isalpha()
.isspace()
.startswith()
.endswith()
```

Extra beginner-friendly string helpers should be considered:

```pr
text.contains("abc")
text.before("-")
text.after("-")
text.reverse()
text.words()
text.lines()
```

# Arrays

Arrays should be mutable, ordered, zero-based, heterogeneous, and allow duplicates.

```pr
let nums = [1, 2, 3]
let mixed = [1, "A", 3.4, true, [19]]
let empty = []
let also_empty = arr[]
```

Arrays should support negative indexing and slicing.

```pr
output mixed[0]
output mixed[-1]
output mixed[1:4]
```

Methods:

```pr
.add(item)
.insert(index, item)
.remove(item)
.pop(index)
.sort()
.extend(iterable)
.clear()
.contains(item)
.index(item)
.count(item)
.reverse()
```

Flexible array operations should be easy:

```pr
let nums = [1, 2, 3]
nums.add(4)
nums += [5, 6]
output nums
```

Map/filter style should be added later:

```pr
let doubled = nums.map(x => x * 2)
let evens = nums.filter(x => x % 2 == 0)
```

---

# Dictionaries

Dictionaries are ordered key-value collections.

```pr
let user = {
    "name": "Alex",
    "age": 20,
    "active": true
}
```

Characteristics:

1. Flexible size
2. Ordered by insertion
3. Fast average lookup
4. Values can be any type
5. Mutable
6. Keys must be unique
7. Nested dictionaries should work

Access:

```pr
output user["name"]
user["age"] = 21
```

Friendly dot access should be considered for string keys that are valid identifiers:

```pr
output user.name
user.age = 21
```

Methods:

```pr
.keys()
.values()
.items()
.erase()
.clear()
.get(key, default)
.has(key)
.remove(key)
```

Example:

```pr
if user.has("email") {
    output user.email
} else {
    output "No email yet"
}
```

---

# Sets

Sets store unique values.

```pr
let numbers = {1, 2, 3, 4}
let empty = set()
let from_array = set([1, 2, 2, 3])
```

Important rule:

- `{}` currently creates an empty set.
- If Prism later wants empty dictionary syntax, use `dict()` for empty dict or decide one clear rule.

Set operations:

```pr
let a = {1, 2, 3}
let b = {3, 4, 5}

output a | b   # union
output a & b   # intersection
output a - b   # difference
output a ^ b   # symmetric difference
```

Methods:

```pr
.add()
.update()
.remove()
.discard()
.pop()
.contains()
.clear()
```

---

# Tuples

Tuples are immutable ordered collections.

```pr
let single = 1,
let point = (10, 20)
let packed = 1, 2, 3
```

Unpacking:

```pr
let a, b, c = (1, 2, 3)
```

Characteristics:

1. Ordered
2. Immutable
3. Allows duplicates
4. Heterogeneous
5. Supports indexing and slicing

Methods:

```pr
.count(value)
.index(value)
```

# Functions

Functions use `func`.

```pr
func greet_user() {
    output "Hello User"
}
```

If a function does not return anything, it returns `null`.

```pr
func add(a, b) {
    return a + b
}
```

Optional type hints should work:

```pr
func get_user_info(str name, int age, bool is_adult) {
    return "Name: {name}, Age: {age}, Adult: {is_adult}"
}
```

Dynamic style should also work:

```pr
func get_user_info(name, age, is_adult) {
    return "Name: {name}, Age: {age}, Adult: {is_adult}"
}
```

Functions should support default arguments:

```pr
func greet(name = "friend") {
    output "Hello, {name}"
}
```

Named arguments should be supported later:

```pr
greet(name: "Alex")
```

Short lambda functions should be supported later:

```pr
let double = x => x * 2
output double(10)
```

# Control Flow

If/elif/else:

```pr
if age >= 18 {
    output "adult"
} elif age > 12 {
    output "teen"
} else {
    output "child"
}
```

While loops:

```pr
let i = 0
while i < 5 {
    output i
    i += 1
}
```

For loops:

```pr
for item in ["apple", "banana", "cherry"] {
    output item
}
```

Range loops should be built in:

```pr
for i in range(1, 10) {
    output i
}
```

Beginner-friendly repeat loops should be considered:

```pr
repeat 5 {
    output "hello"
}
```

Loop controls:

```pr
break
continue
```

# Flexible Syntax Ideas

Prism should allow clear optional shortcuts where they make the language easier.

## Output without parentheses

```pr
output "Hello"
```

## Optional semicolons

Both should be valid:

```pr
let x = 10
let y = 20;
```

## Simple variable shortcut

Possible future syntax:

```pr
x := 10
name := "Alex"
```

## Natural membership

```pr
if "a" in name {
    output "contains a"
}

if user not in banned_users {
    output "allowed"
}
```

## Safe access

Prism should make missing values easy to handle:

```pr
let city = user?.address?.city ?? "Unknown"
```

Meaning:

- `?.` safely accesses nested values
- `??` gives a default if the left side is null or missing

## Easy try/default expression

```pr
let age = int(input("Age: ")) else 0
```

If conversion fails, age becomes `0`.

---

# Error Handling

Prism should support simple try/catch.

```pr
try {
    let age = int(input("Age: "))
    output age
} catch err {
    output "That was not a valid age"
}
```

It should also support simple error throwing:

```pr
throw "Something went wrong"
```

Errors should be readable:

```text
Runtime Error: Cannot divide by zero
File: main.pr, line 4

4 | let x = 10 / 0
              ^
Suggestion: Check that the second number is not zero before dividing.
```

# Classes and Objects Later

Prism should eventually support classes, but keep them simple.

```pr
class User {
    func init(name, age) {
        self.name = name
        self.age = age
    }

    func greet() {
        output "Hello, I am {self.name}"
    }
}

let user = User("Alex", 20)
user.greet()
```

Object literals should also be easy:

```pr
let user = object {
    name: "Alex",
    age: 20,
    greet: func() {
        output "Hello"
    }
}
```

```pr

random
net
http

```

Examples:


# Pipe Operator

To make Prism easier than Python for data flow, add a pipe operator later.

```pr
let result = text
    |> lower
    |> split(" ")
    |> sort
    |> join("-")
```

This means the output of one step becomes the input of the next.

Alternative method-chain style should also work:

```pr
let result = text.lower().split(" ").sort().join("-")
```

---

# Pattern Matching Later

Prism should eventually support match expressions.

```pr
match status {
    "success" => output "Done"
    "error" => output "Failed"
    unknown => output "Not sure"
    else => output "Other"
}
```

This should be easier than long if/elif chains.

# Beginner-Friendly Examples

## Hello user

```pr
let name = input("Name: ")
output "Hello, {name}!"
```

## Guess age

```pr
let age = int(input("Age: ")) else 0

if age >= 18 {
    output "You are an adult"
} else {
    output "You are young"
}
```

## List names

```pr
let names = ["Alex", "Maya", "Sam"]

for name in names {
    output "Hello {name}"
}
```

## Dictionary user

```pr
let user = {
    "name": "Alex",
    "age": 20
}

output "{user.name} is {user.age} years old"
```

## Flexible function

```pr
func say(message, times = 1) {
    repeat times {
        output message
    }
}

say("Hi", 3)
```

# What Makes Prism Easier Than Python

Prism should aim to be easier than Python in these ways:

1. Output can be written as `output "hello"`.
2. String interpolation should work naturally in normal strings.
3. Variables can be dynamic without type pressure.
4. Optional type hints can be added only when wanted.
5. `unknown` gives a built-in third truth state.
6. Safe access with `?.` prevents common null crashes.
7. Default fallback with `??` makes missing values easy.
8. `repeat 5 {}` is easier for beginners than `for i in range(5)`.
9. Pipe syntax makes data transformation easier to read.
10. Error messages should explain the fix, not just the failure.

---

# Final Vision

Prism should feel like a language where a beginner can write:

```pr
name := input("Name: ")
output "Welcome, {name}!"

repeat 3 {
    output "Prism is easy"
}
```

But an advanced user can grow into:

```pr
let users = fs.read("users.json")
    |> json.parse
    |> filter(user => user.active)
    |> map(user => user.name)
    |> sort

output users.join("\n")
```

That is the direction: simple at the start, powerful later, dynamic by default, and easier to read than Python.



Mood-based truth: true, false, unknown
Prism already has unknown, so lean into it.

let answer = unknown
if answer {
    output "yes"
} elif answer is unknown {
    output "not sure yet"
} else {
    output "no"
}

Useful for forms, AI answers, user permissions, incomplete data.

2. Built-in fallback expressions
let name = input("Name: ") else "Guest"
let age = int(input("Age: ")) else 0

Instead of crashing, Prism lets beginners say: “if this fails, use this.”

3. Safe chains everywhere
let city = user?.profile?.address?.city ?? "Unknown"

No long error-prone checks like:

if user and user.profile and user.profile.address { ... }

4. Human-style repeat loops
repeat 5 {
    output "Hello"
}
repeat while connected {
    read_message()
}
repeat until done {
    step()
}

This is easier for beginners than only while/for.

5. Friendly range syntax
for i in 1..10 {
    output i
}
for i in 10..1 {
    output i
}
for i in 0..100 step 5 {
    output i
}

Cleaner than range(0, 100, 5).

8. Built-in when blocks
when temperature > 100 {
    output "Too hot"
}
when user.logged_in {
    output "Welcome back"
}

Could work like reactive conditions later.


9. Easy event syntax
Good for GUI/game scripting later:

on click(button) {
    output "Button clicked"
}
on key("space") {
    player.jump()
}

This gives Prism a nice identity for apps/games.


10. Built-in validation syntax
let age = input("Age: ") must be int else "Enter a number"
let password = input("Password: ") must len(password) >= 8 else "Too short"

Prism could make safe input easier than Python.


11. Built-in table/data tools
let adults = users
    |> where age >= 18
    |> select name, email
    |> sort by name

This would make Prism great for data scripting.


12. “Explain” mode
Prism could have a learning feature:

explain fib(5)

It could show what the program is doing step by step. Great for beginners.


13. Natural assertions
expect age > 0 else "Age must be positive"
expect name not empty else "Name is required"

Cleaner than verbose assertion functions.
